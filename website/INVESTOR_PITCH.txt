1. THE PROBLEM: THE CONNECTIVITY GAP IN UNIVERSAL HEALTHCARE
-----------------------------------------------------------
While 4G/5G covers cities, millions of patients—regardless of age—living in 
rural or semi-urban "grey zones" lack continuous monitoring.
*   Chronic Disease Burden: 70M+ Indians have Diabetes; millions lack monitoring.
*   Post-Surgical Risks: Re-admission rates are high due to lack of home vitals.
*   Telemedicine Failure: Video calls fail without reliable real-time vitals.

2. THE SOLUTION: AYULINK UNIVERSAL
-------------------------------
AyuLink is a platform for ALL users requiring continuous clinical oversight. 
We provide a "Virtual Hospital Ward" experience at home, powered by LoRa Mesh.

Key Pillars:
*   MONITOR: Medical-grade vitals for Chronic and Post-Op patients.
*   DISPENSE: Automated pill delivery for complex medication regimes.
*   TELEMEDICINE: Real-time data feed for remote specialist consultations.
*   RESPONSE: Emergency dispatch for accidents, falls, or sudden vitals drop.

3. TECH STACK: THE ARCHITECTURE OF RELIABILITY
----------------------------------------------
AyuLink combines modern web performance with rugged industrial hardware:

*   Frontend & Analytics: Next.js 16 (React 19), Tailwind CSS, Leaflet.js Maps.
*   Core Engine & Database: Local-First API with JSON persistence (Zero Latency).
*   Communication Layer:
    - Gateway: ESP32-WROOM with LoRa (433MHz) Transceiver.
    - Relay: Python-based Serial-API Hybrid Bridge for 2-way TX/RX.
    - Mesh: Custom peer-to-peer protocol for "Zero-Internet" transmission.
*   End-Nodes (Dispenser): NodeMCU (ESP8266) controlling multi-servo actuators.

4. 2-WAY COMMUNICATION: THE "WOW" FACTOR
-----------------------------------------
Unlike passive trackers, AyuLink is ACTIONABLE.
- Hardware -> Web: SOS button press -> Real-time Siren & Map Focus.
- Web -> Hardware: Doctor clicks "Dispense" -> Physical pill drops in 2 sec.
This loop is powered by our custom [CMD] / [DATA] serial protocol.
